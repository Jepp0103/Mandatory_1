Commands and tools

Node

    Node is an open-source JavaScript runtime environment which can exequte JavaScript code 
    outside a browser. Node.js can let JavaScript become executed in a command line before 
    the actual content is shown in a browser. Node.js is a paradigm to include JavaScript in 
    many frameworks. The js-notation in "Node.js" doesn't refer to a js-file but is just the 
    name of the service to run JavaScript.

How to run a JavaScript file with Node

    At first the user navigates to a specific folder where the given JS-file is located. 
    That can be done by using the cd command and then use the 
    tab button on the keyboard to browse into the right folder. After accessing the right 
    source folder, the given js-program can be run with the following command:

PS C:\Users\Jeppe N. Dyekjær\Documents\3._Datamatiker\Git_projekter_4_semester\Node_JS\Mandatory_1> node app.js

Routing and callback function


    Routing is about how endpoints of applications (URIs) respond to client requests.
    Routing in Express is defined by using certain methods in the Express environment which
    corresponds to HTTP-methods ex. app.get() to handle GET requests, 
    app.post() to handle POST requests or app.all()
    to handle all types of HTTP-methods at once.

    All these routing methods specify a callback function (handler functions), which is 
    called when an application receives a request to a specified endpoint. We make the application
    listen for certain routes and methods in order to detect a match.

    Routing methods can have multiple callback functions as arguments. When having to use
    the next defined callback function after the first, you have to use the method 
    next() to enter that one.

    The over all definition and method call declaration of a route looks like this
    app.METHOD(PATH, HANDLER);

    app: an instance of express.
    METHOD: an HTTP request method, in lowercase.
    PATH: a path to a given server.
    HANDLER: a function executed when the route matches the function.

Express


    Express.js is a sub framework for Node.js. It is a server framework for Node.js which works as a 
    back-end component for MEAN-stack (overall list of components written in JavaScript
    including MongoDB-database, AngularJS, Node.js, Express.js). Express helps optimizing performance
    and does also support SSL and TLS security systems. 
    Express provides the service to send html pages to a browser so they can get displayed.
Setting up the Express web framework for Node.js
                        At first a JSON-file called package.json has to be created with the following dependencies
                        and their version numbers:
                                                    
                            
                        { 
                            "dependencies": { 
                                "express": "4.17.1" 
                            } 
                        } 
                            

                Afterwards you have to install all node modules which are given libraries for the node running tool.
                In order to install the modules, simply type npm install in the terminal after navigating 
                to the folder where the package.json file is located. From there the node tool will generate 
                a package-lock.json file which includes dependencies for the node modules.
                                                
                At last you have to import the express functionality in your wished JavaScript-file in order to use 
                the modules that has been installed. That is done by the following code:
                        
        //The way to import packages from node modules 
        var express = require("express");
        var app = express();
        const request = require("request");

nodemon


        nodemon is a tool that can help to develop node.js based application to restart automatically
        each time some edition to a file has been made. The tool detects a change to file frequently.
        nodemon does not require any editions to code or development. It's an extension for running 
        js-applications.
Setting up nodemon tool
  1. At first install nodemon tool by using npm as a development dependency: 
    npm install -g nodemon
                                    
  2. After the installment the nodemon tool can be used to run a js-program:
    nodemon app.js

Budo


Budo is a server focused reloading integration for js-development. The tool live reloads a working page
every time a change to the document occur. That makes it a lot easier to test visualization of a given
HTML-page for instance.
Setting up the Bundo tool
  1. At first install bundo by running the command npm install budo -g
                                        
  2. Afterwards bundo can be used to live reload a certain file by running the following command after
entering the correct folder ind the terminal:
    PS C:\Users\Jeppe N. Dyekjær\Documents\3._Datamatiker\Git_projekter_4_semester\Node_JS\NODE_JS_2020\3._jQuery\1._Intro_To_jQuery> npx budo index.js --live --open 
    
When the command succeeds with a status 200 get response in the terminal, the file will 
now automatically update in the browser window after each edition to the file.
